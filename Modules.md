# Modules
전역 스코프 분리
캡슐화
재 사용성 높이기
의존성 관리

### 변수, 인터페이스, 클래스 등을 묶어서 캡슐화
- C#/Java의 네임 스페이스/패키지와 같은 개념
- 서브시스템으로 코드를 조직화
- module 키워드로 생성

### Internal 모듈과 External 모듈로 구분됨
- export 키워드로 모듈 내에 선언된 멤버를 모듈 밖에서도 public하게 사용 가능하도록 설정

### 하나의 모듈을 서로 다른 파일로 분리 가능
### 컴파일 후 하나로 통합 가능
### 외부 라이브러리로 사용 가능, 하나의 스코프로 묶음 가능
